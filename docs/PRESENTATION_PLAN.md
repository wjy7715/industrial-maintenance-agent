# 最终项目总结汇报 PPT 计划

PPT 在功能冻结和最终测试完成后制作，避免展示内容与最终实现不一致。目标文件为 `deliverables/工业运维Agent项目总结.pptx`，同时生成 PDF 或逐页图片用于视觉验收。

## 建议结构（12—15 页）

1. 封面：项目名称、角色、时间。
2. 选题背景与真实问题。
3. 用户、场景与产品定位。
4. 需求范围、P0/P1 与安全边界。
5. 产品核心流程。
6. 总体架构与多工具 Agent 协作。
7. 数据、知识与来源治理。
8. 关键功能演示：诊断、证据、工具轨迹和反馈。
9. 只读遥测与维修闭环适配器。
10. 安全设计：失败关闭、风险升级、禁止自动控制。
11. 测试、盲测与影子评测指标。
12. 开发过程、关键决策与问题复盘。
13. 项目成果和可量化交付物。
14. 已知限制与真实试点条件。
15. 下一步路线图与总结。

## 证据要求

- 所有测试数量、性能和评测指标从最终报告读取，不手工猜测。
- 架构图与实际模块一致；页面截图使用最终冻结版本。
- 明确区分仿真、公开、用户导入和真实现场数据。
- 不使用“工业准确率”“已生产落地”等无法证明的表述。

## 视觉验收

- 统一主题、字号、色板、图标和页脚。
- 每页只表达一个核心结论，避免大段代码和密集文字。
- 架构、流程、指标优先使用图表；重要数字附范围说明。
- 生成后逐页渲染检查文字溢出、遮挡、低对比度和中文字体。
