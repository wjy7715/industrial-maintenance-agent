# MRD｜工业运维 Agent 市场需求文档

## 1. 调研结论

本项目有明确的产业方向和用户痛点，但当前仍是桌面研究结论，不应伪造市场规模或客户验证。首期应从“有基础数据、关键旋转设备、专家资源紧张”的中型制造现场开始。

## 2. 外部环境

- 制造业 AI 政策已将工业智能体、工业高质量数据集、典型场景和预测性维护纳入重点方向。[国家数据局等八部门](https://www.nda.gov.cn/sjj/zwgk/zcfb/0112/20260107214358696030895_pc.html)
- 工信部持续强调智能制造、工业互联网和面向场景的数字化转型方案。[工业和信息化部](https://www.miit.gov.cn/jgsj/xxjsfzs/gzdt/art/2025/art_c195b1b6c1f343e392ee83bf960af9f6.html)
- Microsoft 的预测性维护参考架构显示，可靠方案应综合实时 IIoT 事件、维修历史、人员/成本等上下文，而非只靠聊天或单个模型。[Microsoft Learn](https://learn.microsoft.com/en-us/fabric/real-time-intelligence/architectures/predictive-maintenance)

## 3. 市场痛点

| 痛点 | 当前替代方式 | 产品机会 |
|---|---|---|
| 数据与手册分散 | 人工查系统、文件夹、电话询问 | 一次任务中聚合带证据的信息 |
| 专家不足 | 依赖资深人员和微信群 | 将授权知识变为可复核草稿 |
| 告警过载 | 人工筛选阈值告警 | 风险、历史和知识关联 |
| 通用模型不可信 | 只用于写作或禁用 | 工具契约、引用、审计和拒答 |
| 重型平台难快速试点 | EAM/APM 或 Excel | 轻量决策支持与集成入口 |

## 4. 目标用户

### 首选市场

- 拥有离心泵、电机、风机或压缩机等关键旋转设备。
- 已有部分传感器、维修记录和电子手册。
- 专家知识集中，班组或站点间复用困难。
- 不希望首期替换 EAM/CMMS，但愿意接入只读数据。

### 角色

| 角色 | 核心诉求 |
|---|---|
| 维修技术员 | 快速得到有顺序的核查步骤 |
| 可靠性工程师 | 查看状态、历史、来源和冲突 |
| 维修主管 | 风险优先级、审计与复盘 |
| 知识管理员 | 版本、适用型号、审核和失效管理 |
| IT/OT 安全人员 | 最小权限、隔离、日志与可运维性 |

## 5. 竞争格局与定位

| 类别 | 优势 | 本项目的差异化 |
|---|---|---|
| IBM Maximo 等 EAM/APM | 资产生命周期、工单与预测能力完整 | 不替代；作为自然语言决策支持与协同层 |
| Siemens Senseye 等预测维护平台 | 工业经验、跨资产监测 | 先做单设备族、可解释的轻量方案 |
| 云数据分析平台 | 实时数据与建模能力强 | 可作为后续数据平台，不是开箱即用运维助手 |
| 通用 RAG/低代码 Agent | 搭建快 | 强化工具安全、型号适用性、引用和评测 |
| 人工 + 手册 + 表格 | 灵活 | 减少查找成本并标准化排障记录 |

定位：面向具备基础设备数据和维修资料、但缺少可规模化专家支持的运维团队，提供一个带证据、可解释、人工在环的多工具运维 Agent。它不是重型资产管理平台，也不是自动控制系统。

## 6. 市场测算和访谈

本阶段不编造 TAM/SAM/SOM。后续使用自下而上测算：目标企业数 × 单企业年度软件/实施预算，再根据设备覆盖、数据成熟度、销售能力和转化率收缩。

至少访谈维修、可靠性、主管、IT/OT、安全与采购角色，重点询问：最近一次故障处理耗时、查过哪些系统、已有数据与接口、哪些操作绝不能自动化、什么证据会使其采纳、什么指标改善才有采购价值。
