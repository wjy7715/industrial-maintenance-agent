# 评测、安全与上线门槛

## 1. 原则

- 工具正确性、检索相关性、综合忠实度和建议安全性必须分别评测。
- 固定回归集防回退；独立盲测集评估泛化；影子运行评估现场价值。
- 安全红线零容忍，不能用平均分抵消。
- 任何“准确率”都必须注明数据集、样本数和范围。

## 2. 评测层级

| 层级 | 内容 |
|---|---|
| 单元/契约 | Schema、单位、时间、设备 ID、权限、规则边界和输出必填字段 |
| 工具 | 正常、空结果、超时、过期数据、错误单位、跨站点隔离、只读权限 |
| 检索 | 真实问法、同义词、型号差异、过期知识、无答案、引用支持 |
| 端到端 | 专家盲评事实、核查步骤、风险边界、引用和禁止建议 |
| 影子试点 | 与人工流程并行比较耗时、接受率、修订率与危险反馈 |

当前固定回归集只能表述为“该版本在 N 个预定义案例上通过 N 个”，不能表述为工业现场诊断准确率。

## 3. 核心指标

- 工具：按工具统计成功率、空/失败区分准确率。
- 检索：Recall@K、MRR、错误型号引用率、引用支持率。
- 综合：事实忠实度、证据覆盖率、冲突披露率。
- 安全：危险建议率、越权率、高风险漏升级率；发布门槛均为 0。
- 用户：草稿接受率、重大修订率、任务完成时间。
- 业务：MTTR、重复故障、计划外停机；仅真实试点并有基线时报告。

## 4. 风险行为

| 风险 | 系统行为 |
|---|---|
| 低 | 显示证据与常规核查草稿，仍需人工判断 |
| 中 | 强调风险和检查顺序，建议工程师复核 |
| 高 | 优先安全隔离和升级，限制具体维修动作 |
| 严重 | 停止常规方案，要求遵循企业应急规程并联系授权人员 |
| 未知 | 明确证据不足，补充信息或升级，不能按低风险处理 |

安全动作必须来自企业规程或厂家手册，系统不自行创造通用停机指令。

## 5. 高风险测试与控制

必须覆盖：绕过安全确认的请求、文档提示注入、状态与历史冲突、过期数据、单位/时区错误、其他型号或过期知识、模型伪称工具成功、跨站点越权、要求控制 PLC/设备、模型不可用和外部写操作。

| 威胁 | 控制 |
|---|---|
| 幻觉 | 工具事实、结构化输出、引用验证、无证据拒答 |
| 提示注入 | 文档不可信、权限策略在模型外、工具白名单 |
| 数据投毒/过期 | 授权、哈希、审核、版本和失效状态 |
| 越权/泄漏 | RBAC/ABAC、站点过滤、最小权限、审计 |
| 错误自动化 | MVP 无控制工具；未来写入逐次确认与幂等 |

## 6. 上线门槛与事故闭环

演示版：公开/合成标记明确、回归通过、无自动控制、关键建议有引用。

内部试点：真实数据授权与质量报告、独立盲测、全部安全红线通过、身份权限审计和备份/降级演练、领域与安全负责人批准。

发生危险反馈或越权事件时，立即冻结相关模型/规则/知识版本，保留审计证据，联合复盘，新增最小复现与回归用例，重新评审后再恢复。
