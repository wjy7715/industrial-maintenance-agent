# 数据与知识库建设方案

## 1. 目标与现状

目标是让每条建议都能回答：数据来自哪里、适用于哪台设备、何时有效、谁审核过。

| 当前资产 | 用途 | 限制 |
|---|---|---|
| 合成离心泵遥测 | 演示工具调用、异常组合 | 不能代表真实现场分布 |
| UCI AI4I 公开数据 | 数据管道、异常/分类实验 | 不等同于当前离心泵数据 |
| 公开手册/政府资料 | 检索与引用演示 | 不保证覆盖企业型号和 SOP |
| 固定回归样例 | 防止版本回退 | 不代表现场泛化与商业收益 |

`synthetic`、`public`、`enterprise` 数据必须隔离标识，禁止混称为生产数据。

## 2. 真实试点数据清单

### 主数据

- 站点、产线、设备唯一 ID、设备族、厂家、型号、部件层级和额定参数。
- 传感器、单位、量程、工况与关键阈值。

### 遥测与事件

- 指标、值、单位、采集时间、采样频率和质量码。
- 告警、启停、工况切换、维护窗口。
- 对离心泵优先收集振动、入口/出口压力、流量、温度、电流等，最终由型号与专家确认。

### 维修历史与知识

- 故障现象、故障码、确认根因、处理动作、工单、停机时长和复发情况。
- 厂家 IOM 手册、故障码表、企业 SOP、安全规程、已脱敏案例与专家规则。
- 每份资料必须有授权、版本、适用型号、发布日期、失效日期和审核人。

## 3. 数据准入

接入前完成：所有权/授权确认；设备和时间可关联性；单位/时区/频率说明；缺失、重复、漂移和异常检查；故障标签定义；数据分级；保留/删除策略；数据负责人和更新频率。

如果设备、时间和事件无法可靠关联，不得训练或宣传预测能力；仍可做手册检索和流程辅助。

## 4. 统一数据模型

- `Site`、`Equipment`、`Sensor`、`TelemetryPoint`、`AlarmEvent`。
- `MaintenanceRecord`：故障、根因、动作、工单和审核。
- `KnowledgeSource`、`KnowledgeChunk`：来源、版本、适用范围、哈希与状态。
- `RuleVersion`：风险规则与审批记录。
- `AgentSession`：请求、工具轨迹、输出与反馈。

时间使用带时区标准格式，展示时转换为站点时区；单位使用统一词典，含义不明确时禁止自动换算。

## 5. 知识库生命周期

1. 登记来源、授权、版本、适用设备和责任人。
2. 安全扫描、解析结构、保留章节/表格/警告框和页码。
3. 按维修步骤和语义段落分块，避免跨型号或跨安全章节。
4. 建立元数据过滤和关键词基线；需要时增加向量索引。
5. 领域专家抽样核对，在独立检索集通过后发布。

状态为 `draft → reviewed → active → superseded/retired`。只有 `active` 内容进入默认检索；旧版保留审计快照但不用于新建议。

## 6. 检索与预测门槛

检索评测集应覆盖真实问法、同义词、错别字、型号差异、否定表达、过期文档和无答案场景；报告 Recall@K、MRR、错误型号引用率、无答案识别和引用支持率。

引入向量数据库的前提：评测证明关键词/元数据基线不足，且混合检索在独立测试集提升召回而不显著增加错误型号匹配。

升级到预测模型前，必须具备连续真实时序、确认的失效与维修标签、正常/故障工况覆盖、按时间/设备隔离的数据集、与阈值/人工规则的基线对比、误报/漏报成本定义和影子模式验证。

## 7. 数据安全

- 数据按公开、内部、敏感 OT、受限分级；最小采集与最短必要保留。
- 只读账户接入数据源，密钥托管且不写入日志。
- 多站点/多客户按权限隔离，检索前先过滤。
- 外部模型调用先执行脱敏和策略判断；不满足则使用本地模型或禁用增强。
- 删除知识时同步清除索引、缓存和派生向量，并按审计规则保留最小操作记录。
