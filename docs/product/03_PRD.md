# PRD｜工业运维 Agent 产品需求文档

## 1. 产品定义

| 项目 | 内容 |
|---|---|
| 名称 | 多工具调用工业运维 Agent |
| MVP | 工业离心泵故障查询与维修方案草稿 |
| 形态 | Web/内网应用；后续可提供 API 和企业消息入口 |
| 核心边界 | 只读查询、分析、草稿；不自动控制真实设备 |

目标：用户选择设备、描述症状后，系统安全协调多个只读工具，输出可追溯、可复核、带来源的排障草稿。

## 2. 用户故事

- 维修技术员希望快速获得有顺序的核查步骤，减少手册查找时间。
- 可靠性工程师希望查看每项结论使用的状态、历史和手册证据。
- 维修主管希望高风险情况明确升级，避免未经授权操作。
- 知识管理员希望追溯资料的设备型号、版本和页码。
- IT/OT 管理员希望工具权限最小化并可审计。

## 3. 范围

### P0 必须具备

1. 校验设备 ID 与访问权限。
2. 接收自然语言症状和必要上下文；信息不足时追问。
3. 查询当前/最近状态，显示时间、单位和数据质量。
4. 查询相关故障与维修历史。
5. 执行版本化确定性风险规则。
6. 检索适用型号的维修知识并显示来源。
7. 输出风险、事实、候选原因、核查步骤、建议、未知项和引用。
8. 展示工具调用轨迹、失败和降级状态。
9. 收集用户反馈和专家审核结论。
10. 管理知识来源、版本、状态和适用范围。

### P1/P2

- P1：澄清问题、遥测趋势、评测看板、企业认证、CMMS 工单草稿。
- P2：实时流数据、经验证的预测模型、受控通知、企业微信/移动端、多设备族。

### 明确不做

- 自动启停设备、修改 PLC 或工艺参数。
- 自动下发正式工单、采购、付款、人员调度。
- 无证据的确定性故障结论。
- 未授权向外部模型传输企业数据。

## 4. 核心流程

1. 用户选择设备并输入症状。
2. 系统校验设备、时间和关键信息；必要时提出不超过 3 个澄清问题。
3. 总控调用状态、历史、风险与知识工具。
4. 证据聚合器检查新鲜度、单位、型号适用性和来源冲突。
5. 高风险优先输出升级与安全边界。
6. 生成结构化草稿，明确事实、推断和未知项。
7. 用户查看工具轨迹，提交有效/部分有效/无效/危险反馈。

异常处理：遥测失败必须显示失败而非用旧值冒充当前值；历史为空与查询失败必须区分；知识无匹配时不得生成具体维修动作；模型不可用时回退到规则和模板。

## 5. 功能需求与验收

| 编号 | 需求 | 验收标准 |
|---|---|---|
| FR-001 | 设备识别 | 不存在或无权限设备不得查询；同名设备不串站点 |
| FR-002 | 症状采集/澄清 | 保留用户原文；缺少关键字段先追问 |
| FR-003 | 状态查询 | 返回指标、值、单位、采集时间、质量和来源；过期必标记 |
| FR-004 | 历史查询 | 保留记录编号与时间；空结果和失败分开 |
| FR-005 | 风险规则 | 返回等级、触发规则、依据和升级要求；模型不能降低更高风险 |
| FR-006 | 知识检索 | 返回来源、版本、章节/页码、适用范围；停用知识不返回 |
| FR-007 | 多工具编排 | 每次调用记录版本、耗时、状态与错误；敏感值不入普通日志 |
| FR-008 | 方案草稿 | 事实/推断分离；关键建议必须有来源或推断标识 |
| FR-009 | 反馈审核 | 用户反馈不直接修改生产知识；知识发布需审核 |
| FR-010 | 权限确认 | 只读可自动，外部写入逐次确认；MVP 无设备控制能力 |
| FR-011 | 审计 | 会话可回溯输入、工具、规则/知识版本、输出与反馈 |

## 6. 信息架构与权限

排障工作台包含设备/数据新鲜度、症状与澄清、风险卡片、状态与趋势、历史、知识证据、结构化草稿、工具轨迹和反馈。

| 行为 | 维修员 | 领域专家 | 知识管理员 | 管理员 |
|---|---:|---:|---:|---:|
| 查询授权设备 | 是 | 是 | 按需 | 是 |
| 生成草稿 | 是 | 是 | 否 | 按需 |
| 提交反馈 | 是 | 是 | 否 | 否 |
| 审核结果/规则 | 否 | 是 | 协作 | 否 |
| 发布知识 | 否 | 审核 | 是，需流程 | 否 |
| 修改权限/接口 | 否 | 否 | 否 | 是 |
| 控制设备 | 否 | 否 | 否 | 否 |

## 7. 非功能与发布门槛

- 本地确定性链路目标 P95 ≤ 5 秒；模型链路目标 P95 ≤ 15 秒，超时必须降级。
- 关键建议引用覆盖率 100%；未经确认写操作和自动控制为 0。
- 采用最小权限、传输加密、密钥不入日志、角色访问控制和完整审计。
- P0 自动化与人工验收通过；安全红线用例全部通过；独立测试集经专家盲评达到事先约定门槛；完成日志、备份、回滚和故障降级演练后才能试点。
