# 实施计划、里程碑与交接清单

## 1. 执行策略

按“问题验证 → 数据准入 → 安全 MVP → 影子试点 → 受控集成 → 扩展”推进。阶段门优先于时间表；任何阶段未通过，不得提前扩大设备范围或写权限。

## 2. 阶段计划

### 阶段 0：需求与立项验证（1—2 周）

工作：访谈维修、可靠性、主管、IT/OT、安全角色；选择一个设备族和 3—5 个高频故障；盘点数据、手册授权、接口和人工流程基线。

退出：业务、领域、安全和数据负责人明确；场景、数据和成功指标可验证。否则保持教学项目。

### 阶段 1：数据与知识基线（1—3 周）

工作：建立设备映射、单位和时间规范；导入授权知识；完成数据质量、知识版本、检索基线与独立测试集。

退出：设备、数据、知识可可靠关联；专家确认适用范围和高风险规则。

### 阶段 2：安全 MVP（2—3 周）

工作：实现四个只读工具、统一契约、编排、结构化草稿、审计、反馈、权限、注入防护、超时降级和输出验证。

退出：P0 验收、安全红线、专家盲评全部通过。

### 阶段 3：影子试点（2—4 周）

工作：在不影响正式流程的情况下读取真实数据并生成草稿；与人工流程并行记录耗时、修订、接受和危险反馈；每周复盘失败案例。

退出：样本量达到计划、危险建议为 0、价值或改进证据明确，且一线/专家/安全共同认可下一步。

### 阶段 4：受控集成（3—6 周）

工作：企业认证、站点隔离、集中监控；接入 CMMS/EAM 只读数据；可选逐次确认的工单草稿；完成备份、恢复、升级和回滚。

### 阶段 5：扩展

按价值逐个增加设备族；只有满足数据门槛后再评估实时告警和预测模型。每次扩展重新通过知识、测试和安全阶段门。

## 3. 优先级与职责

| 优先级 | 内容 |
|---|---|
| Must | 单设备族、四只读工具、草稿、引用、风险规则、审计、反馈 |
| Should | 澄清、趋势、知识管理、评测看板、企业认证 |
| Could | 工单草稿、企业消息入口、混合检索、多设备族 |
| Won't now | 自主控制、自动正式工单、采购付款、无验证预测 |

| 工作 | 主要负责 |
|---|---|
| 场景与价值 | 业务负责人、产品、领域专家 |
| 数据准入 | 数据负责人、IT/OT、安全 |
| 知识与规则 | 领域专家、知识管理员 |
| 技术实现 | 架构、研发、算法 |
| 评测与验收 | 测试、领域专家、产品 |
| 上线决策 | 业务负责人、安全、IT/OT |

## 4. 首周交接清单

1. 确认真实行业、设备型号和试点性质。
2. 完成至少 5 类角色访谈，记录原始证据。
3. 建立数据源、手册授权和版本表。
4. 定义 3—5 个故障场景、禁止建议和人工流程基线。
5. 冻结 P0、Schema、成功指标、风险等级和安全红线。
6. 建立独立评测集；测试集不得参与调优。
7. 完成数据流图、权限矩阵和威胁模型后，才创建代码迭代任务。

## 5. 最终建议

近期不要继续堆叠 Agent 角色或设备类型。最有价值的下一步是用一个可授权的真实离心泵场景，验证“证据聚合 + 标准化核查”是否比现有人工流程更快、更完整。验证通过后，再在 CMMS 集成、混合检索或预测模型中选择一个方向扩展。
