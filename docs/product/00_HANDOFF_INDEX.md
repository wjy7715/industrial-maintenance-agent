## 已确认基线

- 首个演示对象：工业离心泵；MVP 不覆盖所有工业设备。
- 输入：设备标识、故障现象及可选上下文。
- 工具：状态查询、故障历史、风险规则、维修知识检索。
- 输出：候选原因、核查步骤、处置建议、安全提示、来源和工具轨迹。
- 产品性质：带证据的辅助决策与维修草稿工具，不是自动控制系统，不替代工程师。
- 权限边界：首期只读查询和草稿生成；禁止自动停机、改参数、下发工单、采购和付款。
- 数据边界：现有合成/公开数据仅支持演示与研发，不能证明真实现场准确率。
- 技术边界：确定性规则是安全基线；模型或 Hermes Agent 只能作为可替换增强层。

## 后续开始前必须确认

1. 首个真实试点所属行业、工厂、设备型号和业务负责人。
2. 设备遥测、故障历史、工单和手册是否已获得合法授权。
3. 数据接口、部署位置、网络隔离和数据分级要求。
4. 领域专家、安全负责人和数据负责人是谁。
5. 当前人工排障耗时、质量与停机基线是什么。
6. 哪些建议必须由工程师或安监人员复核。

## 关键决策

| 决策 | 结论 |
|---|---|
| 是否宣称预测性维护 | MVP 不宣称；缺少真实连续时序、失效标签和现场验证 |
| 是否立刻引入向量数据库 | 否；先建立关键词/元数据检索基线与评测集 |
| 是否让模型决定安全等级 | 否；高风险规则与权限策略在模型外执行 |
| 是否支持自动设备控制 | 否；架构层不提供该工具和权限 |
| 是否一次覆盖多设备 | 否；先以离心泵建立可评测模板 |

## 推荐执行顺序

1. 冻结目标设备与真实场景。
2. 完成数据授权、质量审计和专家访谈。
3. 评审 BRD、MRD、PRD、权限矩阵和安全红线。
4. 先建立独立评测集，再实现或扩展 Agent。
5. 完成影子试点后，才讨论 CMMS 写入、实时监控或预测模型。
