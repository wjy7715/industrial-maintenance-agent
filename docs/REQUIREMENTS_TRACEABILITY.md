# 需求追踪矩阵

状态说明：`完成` 表示本地实现和自动化证据已具备；`进行中` 表示已有基础但未达到发布门；`外部依赖` 表示需要真实组织授权，不能由教学项目伪造。

| 需求 | 当前状态 | 已有证据 | 发布前缺口 |
|---|---|---|---|
| FR-001 设备识别与权限 | 本地完成 | 未知设备拒绝；站点/角色策略；越权与不存在统一响应 | 企业身份源与组织目录为外部依赖 |
| FR-002 症状采集与澄清 | 完成 | 空输入拒绝；模糊症状最多 3 个问题 | 增加上下文展示与审计断言 |
| FR-003 状态查询 | 完成 | 数值、单位、量程、采集时间、来源、新鲜度、多点趋势与质量摘要 | 实时传感器质量码为外部依赖 |
| FR-004 历史查询 | 完成 | 独立事件 CSV、告警重放、维修闭环和来源 | 增加更多不一致事件质量提示 |
| FR-005 风险规则 | 完成 | 版本化阈值、标准单位校验、高/严重风险升级与边界测试 | 真实现场阈值仍属外部依赖 |
| FR-006 知识检索 | 完成 | 型号、版本、状态过滤；来源和页码；发布校验与注入扫描 | 扩充正式厂商知识需另行授权 |
| FR-007 多工具编排 | 完成 | 统一契约、状态、耗时、错误和降级 | 增加超时/重试策略的确定性测试 |
| FR-008 方案草稿 | 完成 | 事实/推断/未知/冲突分离；独立验证器检查引用、禁止动作和安全升级 | 专家盲评仍属试点门槛 |
| FR-009 反馈审核 | 本地完成 | SQLite 普通反馈、专家审核状态/结论与身份分表留痕 | 企业审核人目录为外部依赖 |
| FR-010 权限确认 | 本地完成 | 工具白名单、默认拒绝、站点/角色授权、外部写入确认、设备控制禁止 | 企业 IAM 为外部依赖 |
| FR-011 审计 | 完成 | 请求、计划、工具轨迹与反馈可查询 | 发布前备份/恢复演练 |
| P1 遥测趋势 | 完成 | 多点 CSV、最新点选择、连续性提示、趋势摘要和网页图表 | 实时流与长周期存储为外部依赖 |
| P1 评测看板 | 完成 | 回归集、独立盲测、失败分类、影子指标和本机延迟基线 | 真实现场盲评为外部依赖 |
| P1 知识治理 | 本地完成 | 状态、版本、型号、双人审核、引用与注入校验 | 企业发布审批系统为外部依赖 |
| 企业认证/站点隔离 | 外部依赖 | 架构与权限矩阵 | 真实身份源、组织与部署环境 |
| CMMS/EAM 集成 | 外部依赖 | 历史只读适配接口 | 厂商 API、授权和逐次确认流程 |
| 真实 OT/现场试点 | 外部依赖 | CSV 只读导入和影子指标 | 数据授权、现场阈值、专家与安全批准 |

## 本地发布剩余顺序

1. 治理：备份恢复和运行手册。
2. 安全、回归和端到端验收。
3. 发布：统一验收、项目总结 PPT、最终 GitHub 推送。
